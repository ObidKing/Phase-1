

# My Path to LLM Engineering: From Zero to Building Models from Scratch

It *is* possible to learn how to program from scratch and evolve into an LLM Engineer—someone capable of building Large Language Models, understanding the inner workings of GPUs, and writing CUDA code.

While many people are fascinated by using LLMs and watching these models explode across the industry, my goal is different. I don't just want to prompt and enjoy the outputs; I want to go under the hood, build from scratch, and explore what is truly possible.

This repository will document my research and learning path. Today, I start.

## Phase 1: Python Foundations

To progress from zero programming experience to a capable developer, I am using the following resources:

* **CS50P:** Introduction to Programming with Python (Harvard)
* **The Official Python Tutorial**
* **Automate the Boring Stuff with Python (3rd Edition)**

By the end of this phase, I will be able to:

* Design medium-sized Python programs before coding.
* Divide a problem into functions, modules, and classes.
* Confidently use Python's fundamental data structures.
* Read and write CSV, text, and JSON data.
* Build Command Line Interface (CLI) applications.
* Handle errors gracefully.
* Write unit and integration tests.
* Debug unfamiliar code.
* Explain time and space complexity.
* Build reusable game engines.
* Complete standard Python coding interviews.
* Present a finished project.

## Standard Project Structure

Every project I build will follow this strict directory structure:

```text
project-name/
├── README.md
├── DESIGN.md
├── CHANGELOG.md
├── pyproject.toml
├── src/
├── tests/

```

## My Rule for AI Assistance

I will do my best to avoid relying on AI prompts for coding help. Currently, AI tends to just hand you the answer, which shortcuts the learning process. Instead, I will rely on official documentation and direct searches to solve problems.
